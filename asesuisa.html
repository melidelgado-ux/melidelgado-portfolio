Case Study · UX/UI · Webflow

Asesuisa: una web de seguros más clara, humana y escalable.

Migración y rediseño integral de una experiencia web responsive, con foco en claridad de navegación, consistencia visual, design system, CMS y componentes pensados para crecer.

Rol
UX/UI Designer freelance vía Elaniin
Periodo
2025 - actualidad
Herramientas

Foco
Rediseño web, design system, CMS y responsive

Sitio público actual

Referencias visuales de Asesuisa.com

El rediseño todavía está en construcción. Estas piezas corresponden al sitio público vigente y funcionan como referencia visual del ecosistema que se está ordenando.

Contexto

El trabajo comenzó con la migración del sitio legacy a Webflow, proponiendo mejoras sobre el diseño existente. Luego evolucionó hacia un rediseño integral del sitio: home, páginas institucionales y landings por tipo de seguro.

El objetivo fue acercar la marca a las personas, mejorar la orientación del usuario y preparar una estructura visual y técnica que pudiera sostener nuevos productos, planes y contenidos.

Problema

  • La experiencia visual estaba alejada del manual de marca.
  • Había poca jerarquía visual y headers muy similares entre secciones.
  • Algunos componentes funcionaban en desktop, pero se rompían en mobile.
  • La estructura CMS estaba fragmentada en múltiples colecciones.

Proceso

Antes del rediseño se realizó una etapa de análisis, benchmark y levantamiento con stakeholders. Esa etapa permitió identificar patrones útiles, oportunidades de mejora y criterios para justificar decisiones visuales y funcionales.

El trabajo de diseño parte desde Figma, pero pensado desde el inicio para Webflow: componentes posibles de implementar, tokens replicados en ambos sistemas y ajustes reales según las capacidades de la herramienta.

Design system

Definición de colores, tipografías, espaciados, botones, cards, formularios y componentes reutilizables para mantener consistencia entre Figma y Webflow.

CMS escalable

Propuesta para consolidar colecciones de planes y productos, reduciendo complejidad y preparando el sitio para futuras iteraciones.

Resultado parcial

  • Sistema de tokens replicado entre Figma y Webflow.
  • Componentes reutilizables para landings y planes.
  • Mayor alineación visual con el manual de marca.
  • Responsive pensado desde el inicio para desktop, tablet y mobile.
  • Validaciones semanales con cliente y stakeholders.
