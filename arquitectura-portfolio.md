# Arquitectura del Portfolio

## Direccion General

Portfolio bilingue, editorial y navegable, con una estructura clara pero con detalles visuales personales. El hilo conductor es la curiosidad:

"La curiosidad mato al gato, pero el gato murio sabiendo."

La experiencia debe sentirse profesional, humana y curiosa. No debe parecer un template corporativo ni una galeria desordenada. La clave es combinar orden UX con gestos visuales propios.

## Moodboard Inicial

### Referencia 1: Portfolio editorial con detalles manuales

Tomar:

- Tipografia grande y protagonista.
- Fondo claro con mucho aire.
- Detalles dibujados a mano.
- Sensacion de cuaderno, anotacion, exploracion.
- Foto personal tratada como pieza editorial.
- Tono cercano y memorable.

Evitar:

- Que se vea demasiado infantil.
- Exceso de doodles.
- Poca jerarquia en informacion profesional.
- Decoracion que compita con los casos.

### Referencia 2: Wireframe web claro y modular

Tomar:

- Navegacion simple.
- Secciones reconocibles.
- Hero con CTA.
- Cards para trabajos.
- Bloques de skills.
- CTA final de contacto.

Evitar:

- Estetica demasiado generica.
- Grises planos sin personalidad.
- Cards repetidas sin narrativa.
- Layout de portfolio templateado.

### Referencia 3: martapamies.com

Tomar:

- Sensacion profesional y pulida.
- Hero claro con posicionamiento directo.
- Composicion visual lateral con imagenes superpuestas.
- CTA principal visible.
- Navegacion simple.
- Secciones con captions pequeñas y titulos fuertes.
- Enfoque hibrido entre diseño UX/UI y construccion digital.
- Ritmo visual mas sobrio y confiable.

Adaptar a mi identidad:

- Mantener el fondo tipo papel/cuadricula como base propia.
- Usar acentos azul boligrafo en vez de una estetica mas corporativa.
- Incorporar ilustraciones propias en lugar de fotografias genericas.
- Mantener el hilo de curiosidad y aprendizaje.
- Sumar el costado no-code/IA como parte de mi diferencial.

Evitar:

- Copiar estructura o composicion literalmente.
- Perder personalidad analogica.
- Que el portfolio se vea demasiado como plantilla de agencia.
- Que la experiencia se vuelva fria o demasiado corporativa.

## Principios Visuales

- Blanco, negro y neutros como base.
- Un color acento principal inspirado en tinta/lapicera, por ejemplo azul violaceo o indigo.
- Posible segundo acento suave para categorias o estados.
- Tipografia fuerte para titulares, con cuerpo de texto muy legible.
- Detalles manuales como subrayados, flechas, checks, circulos, anotaciones o pequeñas marcas.
- Imagenes reales y capturas de proyectos como protagonistas.
- Cards limpias, no demasiado redondeadas.
- Sensacion editorial: contenido bien compuesto, no saturado.

## Estructura de Home

### 1. Header

Contenido:

- Nombre o logotipo personal.
- Navegacion.
- Toggle ES / EN.
- CTA pequeño: Contact.

Tratamiento:

- Header simple, fijo o sticky suave.
- Fondo claro.
- Detalle manual pequeño, por ejemplo una linea o marca cerca del logo.

### 2. Hero

Objetivo: impacto y posicionamiento.

Contenido ES:

"La curiosidad mato al gato, pero el gato murio sabiendo."

UX/UI Designer con 5 años de experiencia diseñando productos digitales, experiencias web y piezas visuales claras, funcionales y consistentes.

Trabajo entre estrategia, interfaz, comunicacion visual, Webflow y herramientas no-code para transformar contextos complejos en experiencias mas simples de usar.

CTAs:

- Ver proyectos
- Contactarme

Elementos visuales:

- Ilustracion personal o composicion editorial.
- Posible composicion lateral con 2 o 3 piezas superpuestas, inspirada en la referencia de Marta Pamies:
  - ilustracion principal.
  - captura/papel secundario.
  - detalle manual o tarjeta de nota.
- Detalles dibujados: flecha, circulo, subrayado, check, garabato de "curiosity".
- Tipografia protagonista.

Notas:

- La frase puede ser grande, pero el texto profesional debe aparecer inmediatamente para que no quede solo como gesto poetico.
- "Hago alegremente cosas terriblemente serias" queda reservado para About, no para el Hero.
- En ingles usar version mas sutil si hace falta:
  "Curiosity leads the way. Understanding makes the work better."

### 3. Manifiesto Breve

Objetivo: conectar curiosidad con metodo profesional.

Texto ES:

Mi curiosidad no es solo una forma de mirar el mundo. Es una herramienta de trabajo: me ayuda a hacer mejores preguntas, encontrar patrones, ordenar informacion y diseñar experiencias mas claras.

Formato:

- Bloque editorial corto.
- Puede tener 3 bullets:
  - Preguntar mejor.
  - Entender contextos.
  - Diseñar con claridad.

### 4. Featured Work

Cards:

2. Alianza Americas

Layout:

- Asesuisa puede ocupar mas ancho como principal.
- Alianza y Portfolio pueden aparecer en grid secundario.
- Cada card con imagen/captura cuando exista, tags y CTA "Read case".
